# PRD：護理人員自動排班系統

**版本：** 1.0  
**日期：** 2026-04-15  
**作者：** 護理部資訊化專案小組

---

## 1. 專案背景與目標

### 背景

目前護理排班由護理長以手動方式完成，每月需花費 2–3 小時，且容易因疏忽造成排班不均或違反勞工規則（例如連續大夜班過長）。

### 目標

開發一套可在瀏覽器直接執行的排班工具，讓護理長能一鍵產生符合規則的月份班表，並可手動微調。

### 成功標準

- 按下「自動排班」後，30 秒內產生完整的月份班表
- 所有排班規則均自動遵守（不需手動校驗）
- 護理長可透過點擊格子修改個別班次

## 4. 功能需求

### 4.1 年月選擇

- 頁面頂端提供「年份」與「月份」的下拉選單
- 預設值為當前年月

### 4.2 自動排班

- 提供「自動排班」按鈕
- 點擊後，系統依照第 5 節的規則，為每位護理師的每一天分配班次
- 班次共 4 種：`白班`、`小夜`、`大夜`、`OFF`

### 4.3 班表顯示

- 以表格呈現，**行 = 日期（第 1 日至當月最後一日），欄 = 6 位護理師姓名**
- 每個格子顯示該護理師當天的班次文字
- 顏色標示：

  | 班次 | 顏色 |
  |------|------|
  | 白班 | 淺藍色背景（`#DBEAFE`），深藍文字 |
  | 小夜 | 淺橘色背景（`#FED7AA`），深橘文字 |
  | 大夜 | 深紫色背景（`#4C1D95`），白色文字 |
  | OFF  | 淺灰色背景（`#F3F4F6`），灰色文字 |

### 4.4 手動修改班次

- 點擊任意格子，出現下拉選單，選項為：白班、小夜、大夜、OFF
- 選擇後格子立即更新顏色與文字
- 修改後統計表同步更新

### 4.5 統計表

- 班表右側（或下方）顯示統計表
- 欄位：護理師姓名 / 白班天數 / 小夜天數 / 大夜天數 / OFF 天數 / 合計出勤天數
- 即時反映手動修改結果

---

## 5. 自動排班規則

系統在自動排班時**必須**遵守以下規則（優先順序由高到低）：

1. **每日最低人力**：每天至少 白班 2 人、小夜 2 人、大夜 1 人
2. **連大夜上限**：同一位護理師不得連續排大夜超過 **3 天**
3. **每月休假下限**：每位護理師當月 OFF 天數不少於 **4 天**
4. **班次均衡**：在滿足以上規則的前提下，盡量讓每位護理師的白班、小夜、大夜天數接近均等

---

## 7. 技術限制

- **單一 HTML 檔案**（`index.html`），不分多個檔案
- 純 HTML + CSS + JavaScript，**不使用任何外部套件或框架**（不用 React、Vue、jQuery 等）
- **不需後端**，所有邏輯在瀏覽器端執行
- **不需資料庫**，資料只存在記憶體（頁面重新整理後重置）
- 開啟 `index.html` 後即可直接使用，不需安裝任何東西

---

## 8. 不在範圍內（Out of Scope）

以下功能**本次不實作**：

- 使用者帳號與登入
- 資料儲存（重整後班表消失是可接受的）
- 假單 / 特休申請流程
- 列印或匯出 Excel 功能
- 行動裝置 / 響應式設計
- 多病房 / 多單位管理
